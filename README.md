# Generative AI for Zero-Day Attack Detection

This repository contains an academic research project on the use of Generative AI techniques for detecting zero-day cyber attacks.

## Overview

Zero-day attacks are among the most challenging cybersecurity threats because they exploit previously unknown vulnerabilities, making traditional signature-based detection ineffective. This project explores how Generative AI methods such as GANs, VAEs, and LLMs can support the detection of novel threats through anomaly detection, synthetic data generation, and intelligent analysis of cybersecurity data.

## Project Goals

- Examine the concept and risks of zero-day attacks
- Explain the role of Generative AI in cybersecurity
- Analyze the use of GANs, VAEs, LLMs, and XDR-related approaches
- Compare the output of different Generative AI models
- Highlight limitations, risks, and future directions

## Repository Structure

- `report/` – Full project reports in Greek and English
- `assets/` – Figures and visual material
- `prompts/` – Prompts used during the AI-assisted research process
- `docs/` – Short supporting project documents

## Methodology

The project combines:
1. Literature review
2. AI-assisted content generation and analysis
3. Comparative evaluation of Generative AI tools
4. Critical discussion of applicability and limitations

## Key Topics Covered

- Zero-day attacks
- Generative AI fundamentals
- Anomaly detection
- Synthetic cybersecurity data
- GANs, VAEs, LLMs
- Practical limitations of AI-based threat detection
- Comparative evaluation of ChatGPT and Claude

## Limitations

This project is theoretical and research-oriented. It does not implement a production-ready detection pipeline or evaluate real-world cybersecurity datasets experimentally.

## Future Work

Possible extensions include:
- building a prototype anomaly detection pipeline
- evaluating models on cybersecurity datasets
- integrating LLM-based log analysis
- comparing precision, recall, and false positive rates
